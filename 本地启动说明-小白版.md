# 本地网站打不开时，按这个做

你看到“127.0.0.1 拒绝建立连接”，意思是：

网站文件在电脑里，但是“本地网站服务”还没有打开。

你可以把它理解成：网页地址是门牌号，但屋里的灯还没开。

## 第一步：找到项目文件夹

打开这个文件夹：

```text
D:\文档\college boundaries
```

## 第二步：双击启动文件

在文件夹里找到这个文件：

```text
start-local-site.cmd
```

双击它。

你会看到一个黑色窗口。请不要关闭它。

只要这个黑色窗口开着，本地网站就开着。

## 第三步：等待它显示 Ready

黑色窗口里如果出现类似下面的内容，就说明成功了：

```text
Ready
Local: http://127.0.0.1:3000
```

## 第四步：打开网站

在浏览器打开：

```text
http://127.0.0.1:3000
```

如果你已经打开了这个页面，直接点击浏览器的“重新加载”按钮。

## 重要提醒

不要关闭黑色窗口。

如果你关闭黑色窗口，网站就会停止，浏览器就会再次显示“无法访问此站点”。

## 如果还是打不开

请把黑色窗口里的文字截图发给 Codex。

不要只发浏览器错误页，因为真正的原因通常写在黑色窗口里。

## 本地测试和真实上线的区别

本地地址是：

```text
http://127.0.0.1:3000
```

它只在你自己的电脑上能打开。

真正上线到 Vercel 以后，你会得到一个所有人都能访问的公开链接，通常长这样：

```text
https://你的项目名.vercel.app
```
